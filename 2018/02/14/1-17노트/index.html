ProgrammingNotes

1-17노트

2018-02-14
  • Vector content는 늘리고 줄여도 안깨진다.
  • 의미를 가지는 컨텐츠와 장식용 컨텐츠를 분리해라.
  • 표준과 가이드라인을 잘 지키자 (WCAG)
  • tab-index=”0” : 키보드 포커스 못받는 태그한테 포커스를 줄 수 있게 해준다.
  • Nav 태그는 메인 nav에만 쓰는게 좋다.
  • 인식, 운용 이해의 용이성, 견고성
    오른쪽 상단의 버튼을 누르세요 (x)
    오른쪽 상단의 창닫기 버튼을 누르세요 (o)
  • div는 블록을 그룹할때, span은 인라인을 그룹할때 (text나 이미지)
  • 높이값은 어지간하면 고정하지 마라
  • Position : Static 은 마크업된 순서에 따라 배치된다.
  • static은 붙어있는 상태이고 relative는 떠 있는 상태이다.
  • relative 값을 주고 left, top 속성들을 이용해서 static일때의 기본 위치를 기준으로 옮길 수 있다.
  • absolute는 static이 아닌 녀석을 찾을 때까지 부모를 찾는다.
  • CSS에서 태그 선택자를 쓰는 것은 최소한으로 해라.
  • display : flex 는 블록의 성격을 가진다.
  • inline은 블록의 성격이 없고, 너비가 무시되며 컨텐츠에 따라 크기가 지정된다.
  • 코드 상의 공백도 인라인 상자 취급을 한다.
  • 구체성 점수(specificity) : id / class / element
    li.member : 011 / member:010 이면 li.member가 적용된다.
  • li::before 자식요소의 맨 앞에 인라인 상자를 가짜로 만들어준다.
  • 터치 디바이스에서 44px 규칙 (조작 가능한 컨트롤 / 충분한 크기 제공)
  • Carousel의 하단 버튼은 6px 이상
