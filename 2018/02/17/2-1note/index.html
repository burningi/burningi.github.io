2-1note

2018-02-17
  • 콘솔 로그는 자동 개행을 해준다.
  • if(i%2) 조건식이 들어오는 문맥에 boolean 아닌게 들어오면 엔진이 boolean으로 바꿔준다.
  • 문맥을 읽고 형 변환 시켜주는 것을 암묵적 형변환이라고 한다.
  • 생성자 함수는 생성자로 쓰려는 취지 외에는 쓰지 말자(형변환에 쓰지마라)
  • 인식 가능한 대상은 다 객체다.
  • 추상화 : 우리가 관심있는 속성만 프로퍼티로 만든다. 모든 속성을 다 표현할 수 없고 필요도 없다.
  • 프로토타입은 상속을 표현하는 방법이라고 생각해도 된다..
  • 객체의 프로퍼티 이름에 숫자를 쓰면 내부적으로 문자로 바꿔준다.
  • 함수가 객체인 것이 자바스크립트의 장점이다.
  • 메소드는 밑에 두는 것이 관습이다.
  • 객체 리터럴에 의한 객체 생성은 자바스크립트의 큰 장점이다.
  • 객체 생성자함수는 빈 객체를 만든다.
  • first-name : 따옴표 떼면 연산자로 이해한다.
  • 유사배열객체는 숫자를 프로퍼티로 쓴다.
  • 프로퍼티의 동적 생성도 자바스크립트의 특징이다.
  • 가급적이면 순수함수를 추구하여라.
  • 순수함수 / 비순수함수
  • 인스턴스는 원래 클래스 기반 언어에서 클래스로 만들어 낸 것을 의미, 자바스크립트에서는 생성자 함수로 생성해 낸 객체. 관점을 다르게 해서 부르는 것 뿐이다.
  • standard built-in object : 브라우저와 상관이 없을 수도 있는 객체들이다. ????
  • 일급 객체의 특징 4가지
    함수의 반환 값으로 사용할 수 있다.
    함수의 파라미터로 전달할 수 있다.
    무명의 리터럴로 선언할 수 있다.
    변수나 자료구조에 저장할 수 있다.
  • 기명 함수 표현식일때는 변수명으로 호출한다. 기명함수의 이름은 외부에서 볼 수 없다.
  • 결국은 함수 선언식도 변수명을 호출하는 것.
  • 변수 호이스팅
  1. 한 번 쫙 훑을 때 실행컨텍스트의 vo에 변수명을 적는다.
  2. 초기화 (메모리에서 undefined 할당)
  3. 할당은 나중에 할당문 만났을 때
  • 함수 호이스팅
    함수선언식으로 했을 때 1,2,3 한방에 된다.
    함수표현식은 변수 호이스팅을 따른다.
  • 인수, 인자
    인수를 넘겨서 매개변수(인자)에 담는다.
  • 유사배열 객체의 특징은 length가 있다는 것
  • 함수도 length 프로퍼티가 있고 매개변수의 갯수를 의미한다.
  • __prototype__ 은 부모 ‘역할’을 하는 객체이며 부모가 아니다.
  • prototype 프로퍼티는 생성자 함수만 갖고 있는 프로퍼티
  • 프로토타입 없으면 절차지향 언어가 되버린다.
