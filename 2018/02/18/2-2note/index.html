2018-02-18
  • 익명 즉시실행함수 : 결코 한번 이상 호출되지 않는다.
  • callback : 나중에 호출할게
  • [[prototype]]은 스펙이고 그걸 브라우저가 __proto__로 구현하였다.
  • 부모역할을 하는 객체를 프로토타입 객체라고 한다.
  • 기본자료형의 string, number, boolean들은 쩜(.)을 찍는 순간 객체로 바뀐다.
  • 자바스크립트는 진입점이 없다. 앵귤러를 쓰면 시작점이 있다. ??
  • 함수만 지역이 되면 function level scope 블록이 다 지역이 되면 block level scope
  • 전역변수는 브라우저를 끄지 않는 한 계속 메모리를 점유하고 있다. 지역변수를 쓰면 빨리 쓰고 빨리 버릴 수 있다. 이름도 여러번 쓸 수 있다.
  • 변수 찾아가는 일관성 : 가까운 곳부터 찾는다.
  • 찾아 올라갈 때 지역에 없고 전역에도 없으면 전역변수를 만들어 준다.
  • lexical scoping : 호출될 때가 아니라 선언시점에서의 스코프를 갖는다.
  • jquery $ : 변수명이다.
  • npm : node package manager
  • 자바스크립트의 전역은 하나인데 외부에서 구현한 걸 가져다 쓰면 겹칠 수 있다.
  • 타언어는 기본으로 파일 단위의 스코프를 갖고 있어서 파일을 쪼개면 스코프가 나눠진다. 다른 파일에 쓰면 이름을 중복해서 사용해도 괜찮다.
  • 자바스크립트를 어플을 만드는 언어로 쓰려면 전역 문제를 해결해야 한다.
  • 자바스크립트 파일이 다른 자바스크립트 파일을 부를 수 있어야 한다.
  • 웹팩 : 모듈을 가져올 수 있는 것 / 번들러 : 모든 리소스(자바스크립트)를 하나로 만드는 것
  • 웹팩의 단점은 설정이 어렵다. 파슬은 설정이 쉽다.
  • package.json : 패키지들을 관리하는 장부
  • json은 무조건 쌍따옴표
  • 파일명 모듈명 안겹치게 해라
  • 외부 모듈을 가져다 쓴다는 것은 의존성이 있다는 얘기
  • airbnb의 코딩컨벤션이 유명하다.
  • vscode 설정 맞춰놓기
  • error를 던지면(throw) 콜러한테 던진다. 함수가 호출하면 함수한테 던진다.
  • this는 기본적으로 전역 객체를 가리킨다. 단 두가지 경우만 빼고
  1. 메소드 내에 있는 this : 메소드를 호출한 객체
  2. 생성자 내에 있는 this : 생성자 함수가 생성한 객체
  • 메소드 내부에 있는 내부 함수의 this는 전역 객체이다.
  • 전역 변수는 window 객체의 property다.

  • 함수는 window객체의 메소드다.

  • 키워드는 명령어다. (var, new)

  • new를 함수 앞에 붙이면 일어나는 일
  1. 빈 객체 생성
  2. 함수 내부의 this가 생성한 빈 객체를 가리키게 한다.
  3. 프로퍼티 생성 및 할당
  4. 생성된 객체 리턴
  • apply : 앞에 있는 함수를 호출하면서 this를 갈아낀다.
  • es6 ... 조사하기 (3 dots)
