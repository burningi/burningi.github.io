2-5note

2018-02-18
  • api : application programming interface
    인간은 컴퓨터에게 명령을 해야 하는데 컴퓨터는 0,1 밖에 모른다.
    os가 하드웨어를 제어한다.
    모든 프로그래밍 명령은 os한테 부탁하는것이다.
    메모리에 영역을 확보하는 것도 os가 하는 것이다.
    api는 개발자와 os의 접점이다.
  • ui : 사람과 사물 또는 시스템 사이에서 의사소통을 할 수 있게 해주는 물리적, 가상적 매개체
  • native object : 프론트엔드에 관련된 요소(화면, 브라우저)
  • 모든 것은 다 window 객체의 프로퍼티다. (api, built-in object)
  • window(객체), Window(생성자 함수) 다르다.
  • Window가 window를 만들었다.
  • $ 프로퍼티가 제이쿼리다. 다 들어있다.
  • location은 프로토타입에 메서드가 없다.

  • static method : 객체를 생성할 필요 없이 호출할 수 있는 메소드. 프로토타입에 없고 생성자 함수에 들어있다.

  • 백엔드에서는 native object가 필요 없다.

  • 모든 객체의 실체는 다 window객체에 들어가 있다.
  • 전역 객체는 브라우저가 켜지는 순간부터 있어야 한다.
  • eval 쓰지마
  • 2e64 2의 64승
  • 엄청나게 큰 숫자는 about으로 표시한다.
  • 부동소수점
  • 10진수 소수를 2진수를 바꿀 때 제대로 안떨어진다.
  • 0.1 + 0.2 = 오차 생긴다.
  • 자바스크립트는 10 == 10.00 true인데 다른 언어는 다르다. 타입부터 다르다.
  • 대부분의 웹서버는 8080이 기본 포트인데 서버마다 다르다.
  • URI의 #은 hash, # 뒤에는 서버와 통신을 안한다.
  • URI를 2진수화한 문자열이 서버로 전달된다.
  • 아스키코드를 표현할 수 있는 최소한의 비트수가 8비트(1바이트)
  • encodeURIComponent, encodeURI : >>
    encodeURIComponent는 인자 값으로 컴포넌트를 받는 것으로 인식하고 encodeURI는 인자 값으로 URI를 통째로 받는다고 생각한다. 따라서 전자는 ?,=,&을 인코딩하고 후자는 인코딩 하지 않는다.
  • 객체의 실체를 인스턴스라고 따로 부르는 이유는 object도 객체고 Object 생성자 함수도 객체이기 때문
  • Number 생성자는 Object 생성자를 감쌌을 것이다. (추측)
  • 자바스크립트는 try catch로 에러를 처리하기가 곤란하다.
  • 비동기함수는 캐치를 못한다 ????
  • rxjs : reactive 프로그래밍 ???? >>
    모든 데이타를 흐름으로 생각(stream)
  • String 객체는 유사배열 객체이며 for문으로 순회할 수 있다.
  • String은 자바에서는 객체이고 C에서는 참조값이다.
  • 객체인데 배열과 비슷한 객체가 유사배열객체이다.
  • 순회가능하다는 것은 배열의 중요한 특징이다.
  • arguments는 가변인자함수에서 활용이 가능하다.
  • arguments 리스트와 매개변수 리스트가 일치하지 않을 수 있다.
  • arguments 수가 넘치면 알아서 무시한다.
