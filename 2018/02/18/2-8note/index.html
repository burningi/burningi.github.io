2-8note

2018-02-18
  • 브라우저를 이렇게 만들어라 라는 지침이 ecma 표준이다.
  • 지침을 보고 나름대로 구현한 것이 브라우저 엔진이다.
  • v8은 고속으로 실행하기 위해서 c++을 적용한다.
  • DOM은 html을 객체 모델로 만드는 것이다.
  • 조금 빨라야 되는 것은 c++로 되어있다.(DOM)
  • 우리나라는 c++에 대한 수요가 없고 죄다 응용기술이다. (누가 만든 것을 가져다 쓰는것)
  • 삼성의 원천 기술은 램, 휴대폰 os는 안드로이드(구글꺼)
  • 미국은 c++ 수요가 많다.
  • 실행 컨텍스트는 실행 가능한 코드를 형상화하고 구분하는 추상적인 개념이다.
  • 객체는 그냥 참조하고 할당할 뿐이지만 함수는 실행이 가능하다.
  • 타언어의 start는 함수의 호출이다. (ex : c의 시작은 main함수의 호출)
  • 코드의 실행환경을 객체 모델화 해서 관리하는 것이 실행컨텍스트이다.
  • 실행 컨텍스트 stack은 배열로 만들었을 가능성이 크다.(추측)
  • ec: execution context
  • 엄밀히 따지면 자바스크립트는 순수한 인터프리터는 아니다. 약간 하이브리드다.
    글로벌 ec를 먼저 만드는 전처리가 있다.
  • 크롬이 빠른 이유 : 내부적으로는 자바스크립트를 인터프리터 형식으로 안돌린다는 소문이 있다. (내부적으로는 기계어로 떨군다.)
  • js파일이 로딩이 되면 한줄도 실행되기 이전에 전역 실행컨텍스트 객체를 만들어서 실행컨텍스트 스택에 푸시한다.
  • 실행 컨텍스트는 실행이 끝나면 스택에서 팝된다.
  • 함수는 호출이 가능하지만 전역코드는 호출할 수 있는 방법이 없다.
    자기가 알아서 호출되고 매개변수가 없다. vo(variable object)에 arguments가 없다.
  • 전역에 진입하기 전에 윈도우 객체가 없으면 빌트인 객체도 없다. 그래서 윈도우 객체는 있다는 전제를 깐다.
    GO는 원래 있는 상태에서 global EC의 VO가 GO를 가리키게 할 것이라는 얘기
  • 스코프 체인은 array 또는 유사배열객체로 만들었을 것이다.(추측)
  • 전역 컨텍스트의 VO는 GO, 함수 컨텍스트의 VO는 AO
  • GO에는 built-in objects, DOM이 있다.
  • 스코프가 좁아야 빨리 없어진다.
  • 실행컨텍스트의 속박에서 자유로운 변수를 자유변수라고 부른다. (ex : 클로저에서 내부함수가 참조하는 외부함수의 변수)
