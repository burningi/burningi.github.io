프로그래밍

2018-01-23

http://poiemaweb.com/ 의 내용을 정리한 포스트입니다.

프로그래밍 언어

컴퓨터와 인간 모두 이해할 수 있는 약속된 형태의 인공어
프로그래밍 언어는 Semantics(의미)와 Syntax(구문)의 조합으로 표현된다.
프로그래밍은 요구사항의 집합을 분석하여 적절한 자료구조와 함수의 집합으로 변환한 후 그 흐름을 제어하는 것이다.

body 요소의 가장 아래부분에 스크립트 태그를 위치시키는게 좋은 이유

  1. 스크립트의 로딩 지연으로 인해 HTML 요소의 렌더링이 멈추는 것을 방지
  2. 스크립트가 DOM을 조작하는데 로딩이 안되있으면 에러가 발생한다.

브라우저가 스크립트 태그를 만났을때 하는일

  1. 웹페이지(index.html)의 파싱을 중단
  2. 스크립트 파일을 읽고 실행(인터프리터 언어)
  3. 중단된 파싱을 마저 진행

구문(statement)

명령, 구문이 실행되면 무슨 일인가가 일어난다.

키워드

수행할 동작을 규정

변수

메모리상의 주소를 기억하는 저장소이며, 우리가 이해할 수 있는 언어로 지정한다.
의미있는 이름을 써라(Semantic)

변수 명명 규칙

  1. 영문자, underscore(_), 또는 $로 시작한다.
  2. 대소문자 구별한다.

동적 타이핑(dynamic typing)

자바스크립트의 변수는 선언할 때 타입을 지정할 필요가 없고, 값이 할당될 때 자동으로 타입이 결정된다.

변수 호이스팅

호이스팅이란 모든 선언문이 (var, function) 해당 Scope의 선두로 옮겨진 것처럼 동작하는 특성
선언문보다 앞선 위치에서 참조를 해도 참조가 된다. 그러나 값은 undefined
변수는 호이스팅될 때 undefined값으로 초기화 되고
할당문을 만났을 때 비로소 값의 할당이 이루어진다.

var 키워드로 선언된 변수의 문제점

  • 전역변수 남발
  • var 키워드 생략 허용
  • 중복 선언 허용으로 의도하지 않은 변경 가능성이 있다.
  • 호이스팅: 선언하기 전에 참조 가능
    변수의 범위는 좁을 수록 좋다.
    전역변수는 억제해라

일급 객체의 특징

  • 무명의 리터럴로 표현 가능
  • return 값이 될 수 있다.
  • parameter로 넘길 수 있다.
  • 변수나 객체, 배열에 저장할 수 있다.

함수명

함수표현식에서 사용한 함수명은 외부에서 사용이 불가능하다 (자신을 재귀적으로 호출할때만 사용)
