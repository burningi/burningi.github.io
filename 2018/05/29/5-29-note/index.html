소스코드(High Level 언어) => 인간이 인식하는 언어

cpu가 연산을 한다 (실행도 본질적으로는 연산이다.)
소스코드를 기계어로 번역을 해야한다.
번역을 컴파일러가 한다.
VC++2013 컴파일러

C언어 소스코드가 번역될 때
처음엔 helloworld.obj로 바뀐다
최종단계에서는 helloworld.exe가 된다.
obj도 기계어다. (번역된거긴하다)
c언어 소스코드가 설계도라면 obj는 부품이다.
exe는 완성품이다.
설계도 => 부품 : 번역이다.
부품 => 완성품: Link(조립)
번역은 컴파일러가 하고
링크는 링커가 한다.
설계도에서 에러를 수습해야한다.

#include
소스코드 형식은 ~~.c도 있고 ~~.h도 있다.
둘이 용도가 다르다.
.c는 정의용(define, definition)
.h는 선언용(declaration)
.h에 정의 넣을 수 있지만 거의 그렇게 안한다.
Int main (void) 함수 선언 정의
Int : 반환형식
Main : 함수이름
Void: 매개변수
{} : 블록 스코프(변위)
구문(문장) 끝에는 세미콜론이 온다.
절차적 흐름을 기술한다.
여러 항(item)이 모여 구문을 이룬다.
구문이 모여 함수를 이루고
함수가 모여서 프로그램이 된다.
주의할점:
main이라는 함수는 매우 특수한 함수
main함수의 시작이 곧 program의 시작이다
main이 반환됐다= 프로그램이 끝났다.
(컴파일 전처리기 : pre-processor
스코프는 여러 구문을 한 덩어리로 묶을 때 사용

Main () => printf()
