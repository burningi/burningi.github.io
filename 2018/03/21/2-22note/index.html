    요청은 요청하고 끝, 응답을 추적하지않는다, 단방향통신(무전기)
    내가 할말을 하고 끊지 않으면 저쪽이 나한테 말을 못해
    단방향통신의 단점은 request가 response를 모른다는것
    클라이언트가 로그인하고 다른페이지로 이동하면 로그인했는지 안했는지 모른다
    서버와 통신한 이력을 저장하는게 쿠키,세션,토큰
  • http1.1은 html, css, js한번에 못받는다
    html은 완전한 html ()
    기존방식은 불필요한 데이터가 왔다갔다한다.(일부분만 바꿀때 전체 다 로딩)

  • 데이터를 주고받을때 예전에는 xml 요즘은 json
  • json만 보내줄수있는건 아니다. html, css, js도 보낼수있능능력이있다 (response)
  • json을 보내면 결국 html을 만들어야된다
    반대로 만들어서 보낼수도있는데 안하는 이유는 서버에서 일을 적게할수록 돈이 덜든다.

  • JSON Notation 표기법
    자바스크립트 표기법을 따른다
    xml 복잡 가독성 구리고 데이터도 커진다
    json이 표준됐다
    데이터를 담을 포맷을 의미하고 js object와 거의 비슷하게 생겼다.
    기본적으로 문자열이다(순수한 텍스트)
    왜?
    통신방식은 http protocol, 그 방식이 문자열로 데이터를주고받는다 (정확히는 아스키코드로)
    그래서 객체로 들고있는 정보를 문자로 바꿔야한다.
    js에서 property명은 기본적으로 문자열, 따옴표 생략 가능, 홀따,쌍따 다 가능
    json은 따옴표 생략 불가, 반드시 쌍따옴표
    json값에 타입이 있다.(넘버, 불리언, 스트링)
    json은 기본적으로 객체로 변환될 운명인애들
    JSON이라는 객체의 static 함수 stringify
    개행하는것도 바이트수 추가되니까 개행없이 만들어줌
    stringify 3번째 매개변수는 indent 간격 수
    replacer함수는 필터링함수
    객체에서 뭘좀빼고보낸다(변형하면서 보낸다)
    stringify는 왜 배열에서도 동작할까?
    객체안에 배열이 들어갈 확률이 높기때문에 제공한다.
