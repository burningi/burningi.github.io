  • 타입 지정이 좋다
  • type을 ide, editor가 알수있게 해주면 도움을 많이 받는다.
  • 리얼타임 에러감지
  • flash, silverlight, activex를 js가 대체하게 되었다.
  • es6 는 모듈기능이 아직도 안되고있다.
  • es6 클래스 기능이 모자라다.
  • coffeescript, dart, haxe 컴파일하면 js된다.
  • vscode랑 typescript를 같이 만들었다. ts가 뜨면서 vscode가 같이 뜬다.
  • ts는 es6,7,8 다안다. + 추가기능 추가시킴, es5 써도 돌아가긴한다 ts에서
  • ts는 내부에서 트랜스파일링함, 바벨 쓸필요없고 import export 다 지원한다.
  • angular는 ts베이스에서 돈다.
  • ts 버전업을 따라가면 next es에 대한 추가적인 공부가 필요없다.
  • ts쓰면 개발자의 의도가 코드에 드러난다. 리딩이 쉬워진다.
  • ide, editor 구분 어렵다
  • 바벨은 하위버전으로 js를 트랜스파일링하기 위해서
  • 웹팩 : 모듈로더, export, import / 컴파일,, 트랜스파일 한번에 가능,(태스크매니저) / 여러개의 js, css, image, txt파일들을 하나로 뭉쳐준다. 모듈 번들러
  • ts는 모듈로더의 기능이있다.
  • 근데 브라우저에선 어차피안됨
  • 브라우저에서 안쓰고 노드환경에서 쓸거면 그냥 쓰면됨
  • ts는 amd방식으로 모듈로더기능구현해놈

  • 자바는 무조건 파일하나당 클래스하나

  • enum은 상수들의 집합체를 가독성 높게 할당하려고

  • 자바도 런타임에 발생하는 에러가 있다. 정적 타입이 우수하다고 말하는 이유는 런타임 이전에 에러를 검출한다는 이유인데 백퍼센트 걸러내지는 못한다.
  • 접근제한자 안붙이면 ts는 기본이 public, java는 protected
  • public으로 공개된 것들은 사용자가 알아야한다. 쓸데없이공개하면 몰라도되는것을 알아야한다.
  • 생성자함수에 인자여러개 넘길때 객체로 만들어서줘라
  • 앞에 const를 못붙여서 readonly 생김
  • class는 타입의 이름으로도 쓰인다.
  • 인터페이스를 implements한 클래스로 만든 인스턴스의 타입은 클래스의 타입도 되고 인터페이스의 타입도 된다.$$$$
