2018-04-30
  • es6 모듈과 angular 모듈은 다르다
    export, import 개념을 포함한다
  • decorator metdata는 앵귤라에게 주는 정보
  • 브라우저 모듈은 루트모듈이 반드시 import, 루트모듈 아닌 다른 모듈은 반드시 commonModule import
  • 기능 모듈 하나가 하나의 화면
  • ajax를 쓰게된이유
    웹어플이지만 웹아닌것처럼 쓰고시;ㅍ다
    브라우저는 웹페이지를 표시하라고만들어준거고 어플돌리라고만든게아닌데
    데탑에서 돌아가는 어플을 브라우저에서 돌리고 싶더라
    깜빡이지않고 데탑에서 쓰는것과 거의 동일한 ux를 보장하면서 쓰고싶더라
    ajax 를 사용하면 안깜빡인다
    처음 로딩할때 html하나와 js통째를 싹다가져와서 그 이후에는 요구안함
    서버에서 요청할 자료들만 가져ㅇ와서 클라이언트 사이드에서 html을 만든다
    이렇게 되면 프론트엔드 개발이 복잡해진다
  • ajax는 주소창의 주소를 변경시키지 않는다.
  • SEO가 문제다
    SPA를 웹싸이트에쓰면 검색이안됨
  • 해시는 원래 패이지 내에서 이동하라고 만들어준거 # 들어가면 뒤가 아무리 변해도 안깜빡인다

  • /#/abc

  • 쿠키 : 파일에 써놓자

  • 세션: 서버의 메모리에 써놓자
  • local storage : 브라우저가 갖고 있는 조그만 메모리
    쿠키가 쓰기 불편해서 얘를 쓰지만 장단점이 있다.
    local storage/cookie 둘다 토큰을 저장할 수 있다
  • 로그인회원가입은 엥간하면 전문업체꺼를 써야…보안 다뚫린다.
  • auth0 대표적으로 여기
  • 원래 토큰은 유효기간이 있다.
  • uid, accesstoken을 받아서 백엔드에 줘야된다.
  • submit하기 전에 validate해야한다
  • validators.pattern, validators.required : 에러메시지 여러개 보여줄라고
  • input 태그의 name property를 키로 해서 form이 들고있다
