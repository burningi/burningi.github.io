2018/2/26 note

2018-04-29
  • 핫한 기술에 빨리 접한 사람들이 네임드가 된다.
  • js는 non-block level language
  • function 붙으면 지역변수 떨어지면 전역변수
  • 전역변수는 코딩측면에서(의도치않게 변수명 중복 가능성 있다.)
  • 전역변수를 쓰면 namespace를 오염시킨다
  • var x 선언하면 x선언 한 사람말고 다른사람들이 다 신경써야한다.
  • js는 중복선언을해도 컴파일에러가 발생안하고 의도치않게 변수를 변경할 가능성이 있다.
  • 변수 호이스팅 :
  • var keyword의 약점 : 전역변수 (function level scope만 지원하고 block level을 지원안하기 때문)
  • for문은 위험하다.
  • let : 블록 레벨 스코프 지원
  • 호이스팅이 안되는것처럼 움직인다.
  • 전역변수는 윈도우 끌때까지 계속 살아있다. 전역변수는 메모리를 계속 차지하고있다.
  • scope는 적을수록 좋다. 함수가 만줄이야, 만줄의
  • 함수를 한줄로 쓰는게 트렌드다.
  • 모든 선언문은 호이스팅을 한다
    밑에서 선언되고 상단에서 옮겨진것처럼 동작
  • var
  • 실행컨텍스트 상에는 있지만 메모리엔 없다.
  • let은 호이스팅이
  • for (let i =&&)
  • let 쓰면 자유변수됨 (for문에서)
  • 숫자 쓸일 있으면 상수화 할일 있나없나 체크해야됨
  • _ 열거형이 쓰고싶으면 타입스크립트를 써라
  • const를 쓸때는 ㄲ꼭
  • 객체type은 의도적으로 재할당 해줘야 되는 객체에 대해선 ㅇㄹ반const 써라
